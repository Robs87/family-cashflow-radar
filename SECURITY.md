## 报告漏洞

如果你发现安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下方式私下报告：

1. **GitHub Security Advisories**（推荐）：访问仓库页面 → Security → Report a vulnerability
2. **邮件**：发送至仓库所有者的 GitHub 关联邮箱

### 报告内容

请包含以下信息：

- 漏洞描述
- 复现步骤
- 潜在影响范围
- 建议的修复方案（如有）

### 响应时间

- 确认收到报告：48 小时内
- 初步评估：7 天内
- 修复发布：根据严重程度，30 天内

## 安全最佳实践

### 数据安全

- **不要提交真实账本数据**：`data/raw/` 和 `data/processed/` 目录已被 `.gitignore` 忽略
- **不要提交凭证**：Token、密钥、密码等敏感信息不得写入代码或配置文件
- **使用环境变量**：BeeCount Cloud 的 access token 和 refresh token 应通过环境变量传递

### 本地部署安全

- Web 仪表盘默认监听 `127.0.0.1:8000`，仅本地可访问
- 如需远程访问，请使用反向代理并配置适当的身份验证
- 定期备份 SQLite 数据库文件

### BeeCount Cloud 集成安全

- 默认使用只读权限，不向 BeeCount Cloud 写入数据
- Token 存储在本机 Keychain 或环境变量中，不写入仓库
- 定期轮换 access token 和 refresh token

## 已知安全限制

- 本项目为本地家庭财务分析工具，设计上不面向多用户或公网部署
- Web 仪表盘无内置身份验证，依赖本地网络隔离
- SQLite 数据库文件未加密，依赖文件系统权限保护
