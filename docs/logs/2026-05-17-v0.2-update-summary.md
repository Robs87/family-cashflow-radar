# v0.2 更新摘要

## 背景

v0.1 的主线是：

```text
貔貅 CSV 导入 -> SQLite -> 标准化 -> 分类 -> 月度现金流 -> Web 仪表盘
```

这个方向可以还原历史账本，但容易让产品停留在“报表工具”。用户实际需要的是消费行动前的决策依据：提前还贷、买车、大额消费、投资仓位、家庭安全垫和每月支出控制。

## v0.2 定位

v0.2 将项目主线调整为：

```text
日常记录入口 -> 固定支出自动化 -> 债务/房贷计划 -> 现金流安全判断 -> 近期行动建议 -> 大额决策模拟
```

核心定位：

> 家庭现金流雷达是本地家庭现金流行动建议系统，不是普通记账 App，也不只是 CSV 报表。

## 关键决策

1. Web 首页第一优先级是“一句家庭现金流安全判断和近期建议”，图表是证据，不是主角。
2. 记录入口同时支持对话式记录和 Web 手动输入，二者必须复用同一套新增交易逻辑。
3. 记录颗粒度按家庭现金流语义设计，不只按“收入/支出”粗分。
4. 工作垫付和报销必须单独建模，避免误判为生活支出或真实收入。
5. 房贷、宽带、电话费、保险等确定性账单应自动生成，不要求每月手动记录。
6. 房贷必须拆本金和利息；提前还贷作为一等事件保存、可编辑、可重算。
7. 投资建议先只判断“可动用现金上限”，不做收益预测和复杂投资策略。

## 记录颗粒度

v0.2 最小可用类型：

- 稳定收入
- 日常支出
- 固定刚性支出
- 债务还款
- 投资流入/流出
- 内部转账
- 工作垫付/报销
- 大额资产/一次性事件

这套颗粒度的目标不是做会计分类，而是让系统能判断这笔钱是否影响家庭真实现金流安全。

## 首页建议示例

```text
当前家庭现金流：观察状态。未来 30 天不建议新增 3 万元以上大额支出；可以继续正常消费，但提前还贷建议等 6 月工资到账后再评估。
```

## 对开发计划的影响

后续开发优先级应从“继续堆报表”调整为：

1. 首页建议引擎。
2. Web 手动记录入口。
3. 固定账单和房贷计划。
4. 提前还贷模拟器。
5. 大额消费/买车模拟器。
6. 投资仓位安全上限。
7. 月度支出控制提醒。

历史 CSV 仍有价值，但主要用于校准历史基线和补充资产/债务事件，不再作为每天使用系统的唯一入口。
