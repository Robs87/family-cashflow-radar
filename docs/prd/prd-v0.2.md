# 家庭现金流雷达 PRD v0.2

## 1. 产品定位

家庭现金流雷达 v0.2 是一个本地家庭现金流行动建议系统。

它不是普通记账 App，也不只是把貔貅 CSV 做成报表。它的核心任务是：在用户提前还贷、买车、大额消费、调整投资仓位和控制月度支出之前，判断家庭现金流是否安全，并给出近期行动建议。

一句话：

> 每天记录家庭现金流，自动生成固定支出和债务计划，在消费、投资、提前还贷、买车之前，告诉用户现在安全不安全、近期该不该动钱、最多能动多少钱。

## 2. v0.2 要解决的问题

v0.2 聚焦 6 类决策：

1. 提前还贷：现在还、下月还、暂缓还分别对安全垫和利息有什么影响。
2. 买车：购车首付、贷款和后续固定支出会不会把现金流压到危险线。
3. 大额消费：某笔计划支出能不能做，做完最低现金余额是多少。
4. 投资仓位：哪些钱不能动，当前最多可投入多少而不破坏安全垫。
5. 家庭安全垫：当前现金还能覆盖几个月固定支出和债务还款。
6. 每月支出控制：本月哪些可控支出偏高，近期应该压哪一类消费。

## 3. 首页第一句话

Web 首页最重要的内容不是图表，而是一句可行动判断。

示例：

```text
当前家庭现金流：观察状态。未来 30 天不建议新增 3 万元以上大额支出；可以继续正常消费，但提前还贷建议等 6 月工资到账后再评估。
```

这句话必须同时包含：

- 当前安全状态：safe / watch / tight / danger。
- 最近时间窗口：7 天、30 天或未来 3 个月。
- 明确建议：可做、暂缓、控制、等待某笔收入、最多可动用金额。
- 风险原因：固定支出、债务还款、现金余额、未知分类、异常消费等。

## 4. 记录入口

v0.2 支持两个入口，但底层必须进入同一套交易写入逻辑。

### 4.1 对话式记录

用户可以直接发自然语言：

```text
今天午饭 68
工资到账 20000
帮公司垫付机票 3200
公司报销 3200 到账
买基金 5000
提前还房贷 50000
```

系统解析后写入原始记录、标准化交易、分类结果，并刷新月度现金流和建议。

### 4.2 Web 手动输入

Web 端提供轻量表单，减少高频记录时的 token 消耗。

推荐入口按钮：

- 收入
- 支出
- 转账
- 投资
- 债务
- 垫付/报销
- 大额资产

Web 输入和对话输入不得形成两套账。两者都应该复用同一个新增交易服务。

## 5. 记录颗粒度

记录颗粒度不按普通记账软件的细分类目设计，而按家庭现金流语义设计。

核心判断问题：

> 这笔钱会不会影响家庭真实现金流安全和下一步行动？

### 5.1 最小可用类型

v0.2 最少支持 8 类：

1. 稳定收入：工资、长期项目款、稳定经营收入。
2. 日常支出：餐饮、购物、交通、娱乐等可控消费。
3. 固定刚性支出：房贷、宽带、电话费、保险、学费、水电气物业。
4. 债务还款：房贷、车贷、信用贷、提前还贷。
5. 投资流入/流出：基金、股票、USDT、理财申购和赎回。
6. 内部转账：银行卡互转、信用卡还款等不计入真实收支的流转。
7. 工作垫付/报销：临时占用现金，但不计入家庭消费或收入。
8. 大额资产/一次性事件：买车、卖车、买大件、卖二手、一次性奖金。

### 5.2 工作垫付和报销

工作垫付和报销必须单独建模，不能混入普通支出或普通收入。

示例：

```text
帮公司垫付机票 3200
```

应理解为：

```text
cashflow_direction = outflow
financial_type = reimbursable_expense
affects_living_expense = false
expected_reimbursement = true
```

示例：

```text
公司报销 3200 到账
```

应理解为：

```text
cashflow_direction = inflow
financial_type = reimbursement_income
affects_income = false
linked_to_reimbursable_expense = true
```

如果不这样处理，系统会把垫付误判为生活支出暴涨，把报销误判为收入增加，进而给出错误消费建议。

## 6. 固定支出和债务计划

确定性支出应该自动生成，不要求用户每月手动记录。

v0.2 优先支持：

- 房贷：自动生成还款计划，并拆分本金和利息。
- 提前还贷：作为一等事件保存，可修改，可重算后续计划。
- 宽带、电话费、保险等固定账单：按模板自动生成。
- 生成后的计划以清单方式展示，修改通过单独按钮进入编辑区。

## 7. 建议引擎

建议引擎先做规则化判断，不做黑箱 AI 理财建议。

最小输出：

- 当前安全状态。
- 当前现金安全垫月数。
- 未来 30 天和未来 3 个月风险。
- 本月可控支出是否超预算或超历史基线。
- 当前是否适合提前还贷。
- 当前是否适合买车或新增大额消费。
- 当前最多可投资金额的保守上限。

建议必须可解释，至少说明：

- 用了哪些收入、固定支出、债务还款和现金余额。
- 哪些交易仍是 unknown / pending，导致建议可信度下降。
- 哪个月是风险最高月份。

## 8. v0.2 功能优先级

推荐实施顺序：

```text
1. 首页建议引擎：安全/观察/偏紧/危险 + 一句近期建议
2. Web 手动记录入口：复用对话式 add_transaction 逻辑
3. 固定账单和房贷计划：自动生成，房贷拆本金和利息
4. 提前还贷模拟器：现金流压力 + 节省利息 + 安全垫变化
5. 大额决策模拟器：输入计划支出，输出能不能做和风险月份
6. 投资仓位建议：先判断可动用现金上限，不做投资策略预测
7. 月度支出控制：识别本月高频可控支出是否偏高
```

## 9. v0.2 成功标准

v0.2 成功不是图表更多，而是用户每天打开 Web 后能立刻知道：

1. 家庭现金流现在安全不安全。
2. 未来 30 天有没有需要避开的消费风险。
3. 本月还能不能做大额消费。
4. 现在适不适合提前还贷。
5. 当前最多能拿多少钱去投资而不影响安全垫。
6. 哪类日常支出正在拖累本月结余。

如果系统只能展示流水和图表，但不能给出上述行动判断，就不算达到 v0.2 目标。
