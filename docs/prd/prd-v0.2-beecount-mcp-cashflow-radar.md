# 家庭现金流雷达 PRD v0.2 - BeeCount Cloud MCP 事实源

## 1. 产品定位

家庭现金流雷达不是记账 App，也不替代 BeeCount Cloud。

它是一个运行在 BeeCount Cloud 之上的家庭现金流分析、计划和决策系统：

```text
BeeCount Cloud：记录已经发生的家庭流水。
家庭现金流雷达：管理还没有发生、但已经知道或可以推演的未来现金流，并把它和 BeeCount 流水合并分析。
```

BeeCount Cloud 的职责是事实记录：

- 已发生交易流水；
- 账户；
- 分类；
- 标签；
- 预算等基础账本数据；
- 通过 MCP 向智能体提供账本读取能力。

家庭现金流雷达的职责是计划、预测和决策：

- 通过 BeeCount Cloud MCP 调取已发生流水；
- 维护 BeeCount Cloud 不具备的房贷计划、固定义务、提前关联记录和未来计划事件；
- 生成未来 3 / 6 / 12 个月家庭现金流预测；
- 识别现金流风险、异常支出和预算压力；
- 支持大额决策模拟，例如提前还贷、买车、教育支出、装修和投资。

一句话：

> BeeCount Cloud 负责“发生了什么”；家庭现金流雷达负责“接下来会怎样，以及现在该怎么决策”。

## 2. 不做什么

为了避免重做 BeeCount Cloud，家庭现金流雷达明确不做：

- 不重新实现日常记账录入；
- 不重新实现账户、分类、标签、预算的基础管理；
- 不替代 BeeCount Cloud 的 Web 记账界面；
- 不把 AI 自动分类作为主路径；
- 不自动连接银行、微信、支付宝；
- 不在未确认情况下修改 BeeCount Cloud 事实流水；
- 不在第一阶段做复杂移动端 App。

## 3. 第一阶段目标

v0.2 第一阶段只解决 6 个问题：

1. 能通过 BeeCount Cloud MCP 读取指定账本的已发生流水。
2. 能在本项目中维护未来计划型现金流事件。
3. 能维护房贷 / 贷款计划，并自动展开未来每月应还记录。
4. 能把 BeeCount 已发生流水和未来计划事件合并成现金流时间线。
5. 能输出未来 3 / 6 / 12 个月现金流风险。
6. 能回答一个大额决策是否可执行、风险最高月份在哪里、需要预留多少钱。

## 4. 核心数据边界

### 4.1 已发生流水

已发生流水来自 BeeCount Cloud MCP。

家庭现金流雷达可以建立本地只读缓存，用于：

- 降低重复 MCP 查询成本；
- 做时间序列分析；
- 做规则分类、异常检测和预测；
- 保留分析快照。

但 BeeCount Cloud 仍是已发生流水的事实源。本地缓存不得成为新的记账事实源。

### 4.2 未来计划事件

未来计划事件由家庭现金流雷达维护，代表“还没有发生，但应该纳入现金流预测”的项目。

典型事件：

- 下月工资或项目款预计到账；
- 每月房贷；
- 保险缴费；
- 学费；
- 物业费；
- 车险；
- 计划中的大额支出；
- 计划中的提前还贷；
- 已知未来一次性收入或支出。

### 4.3 房贷 / 贷款计划

BeeCount Cloud 可以记录实际还款交易，但不负责贷款模型。

家庭现金流雷达需要维护贷款计划，包括：

- 贷款名称；
- 贷款类型，例如房贷、车贷、消费贷；
- 初始本金；
- 当前剩余本金；
- 年利率；
- 还款方式；
- 起止日期；
- 每月还款日；
- 每月应还金额；
- 本金 / 利息拆分；
- 提前还款计划；
- 利率调整记录；
- 与 BeeCount 实际还款流水的匹配状态。

### 4.4 提前关联记录

提前关联记录用于解决“未来应发生事项”和“实际 BeeCount 流水”之间的对应关系。

例如：

```text
计划事件：2026-06-20 房贷应还 5000 元
实际流水：BeeCount 里 2026-06-20 支出 5000 元，分类为房贷
系统状态：已匹配
```

匹配后，预测系统应避免重复计算：

```text
同一笔房贷不能既作为计划支出，又作为已发生支出重复进入现金流。
```

## 5. 核心模块

### 5.1 BeeCount MCP 读取层

职责：

- 配置 BeeCount Cloud MCP 连接；
- 读取账本列表；
- 读取交易流水；
- 读取账户、分类、标签等辅助数据；
- 支持按日期范围增量同步；
- 保留 BeeCount 原始交易 ID，确保幂等。

### 5.2 本地分析缓存层

职责：

- 缓存 BeeCount 已发生流水；
- 标准化金额、日期、方向、分类等字段；
- 支持重复同步不重复落库；
- 所有金额统一使用整数分；
- 保留原始 BeeCount payload 方便追溯。

### 5.3 计划事件层

职责：

- 维护未来一次性事件；
- 维护周期性固定义务；
- 支持收入、支出、内部流转、中性事件；
- 支持置信度：确定、较可能、假设；
- 支持启用 / 停用；
- 支持与 BeeCount 实际流水匹配。

### 5.4 房贷 / 贷款计划层

职责：

- 维护贷款基础参数；
- 自动生成未来还款计划；
- 计算本金、利息、剩余本金；
- 支持提前还款模拟；
- 支持利率变动；
- 生成预测现金流事件；
- 与 BeeCount 实际还款流水做匹配。

### 5.5 现金流预测层

职责：

- 合并 BeeCount 已发生流水和未来计划事件；
- 生成按日 / 按月现金流时间线；
- 输出未来 3 / 6 / 12 个月风险；
- 计算现金安全月数；
- 找到现金流最低点；
- 识别压力月份。

### 5.6 AI 分析层

AI 只做解释、分析、提醒和建议，不静默修改事实源。

AI 可以：

- 总结本月现金流；
- 解释结余变化；
- 发现异常支出；
- 推测可能的固定支出；
- 提醒计划事件需要确认；
- 生成决策建议；
- 给出风险原因和行动建议。

AI 不可以：

- 未经确认直接修改 BeeCount Cloud 流水；
- 未经确认直接新增或删除计划事件；
- 把推测当作事实；
- 输出无法追溯到底层流水或计划事件的结论。

## 6. 核心页面 / 交互

第一阶段可以先 CLI 或简单 Web 页面，不追求复杂界面。

### 6.1 现金流首页

展示：

- 当前现金余额；
- 本月已发生收入 / 支出；
- 本月剩余确定支出；
- 未来 3 个月风险等级；
- 现金安全月数；
- 压力最大月份；
- 需要确认的计划事件。

### 6.2 计划事件管理

展示和维护：

- 未来收入；
- 固定支出；
- 大额支出；
- 周期性义务；
- 与 BeeCount 实际流水的匹配状态。

### 6.3 房贷 / 贷款计划

展示：

- 每笔贷款剩余本金；
- 每月应还；
- 本金 / 利息拆分；
- 未来还款时间线；
- 提前还款模拟结果。

### 6.4 决策模拟器

输入：

- 决策名称；
- 金额；
- 发生时间；
- 一次性 / 分期；
- 是否影响收入；
- 是否可撤销。

输出：

- 是否建议执行；
- 执行后最低现金余额；
- 执行后现金安全月数；
- 风险最高月份；
- 需要推迟或减少的支出。

## 7. 风险等级

现金流风险等级：

- `safe`：未来现金流安全；
- `watch`：需要观察，但暂不影响主要支出；
- `tight`：偏紧，需要调整部分计划；
- `danger`：存在断流或刚性支出无法覆盖风险。

所有风险结论必须能追溯到：

- BeeCount 已发生流水；
- 计划事件；
- 房贷 / 贷款计划；
- 预测假设。

## 8. MVP 成功标准

第一阶段成功不是页面好看，而是能稳定回答：

1. BeeCount 已发生流水是否被正确读取和缓存。
2. 下个月有哪些已经知道的固定支出。
3. 房贷未来 12 个月每月要还多少。
4. 已发生流水和未来计划合并后，未来 3 / 6 / 12 个月是否安全。
5. 哪个月现金流最紧，原因是什么。
6. 如果新增一个大额决策，现金流是否会进入 `tight` 或 `danger`。
7. 哪些预测项还只是推测，需要用户确认。
