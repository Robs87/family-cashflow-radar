# 家庭现金流雷达 PRD v0.1

## 1. 产品定位

**家庭现金流雷达** 是一个基于多年貔貅记账 CSV 的本地家庭财务决策系统。

它不是普通记账 App。用户已经有貔貅记账，重复做流水录入没有价值。

它要做的是：

- 家庭现金流仪表盘
- 债务压力监控器
- 大额支出决策模拟器
- 家庭财务安全预警系统

一句话：

> 基于多年账本，自动识别家庭收入、刚性支出、债务、资产事件和未来现金流风险，辅助买车、投资、还债、教育支出等大额决策。

## 2. 第一阶段目标

v0.1 只解决 5 个问题。

### 2.1 真实每月收入是多少

区分：

- 稳定收入：工资、稳定项目款、长期经营性收入。
- 一次性收入：奖金、年终奖、二手出售、临时收入。
- 非收入资金流入：借入、投资赎回、内部转账、退款。

### 2.2 每月刚性支出是多少

重点识别：

- 房贷
- 车贷
- 家庭基础生活费
- 教育支出
- 保险
- 水电气物业
- 通讯
- 必要交通
- 必要医疗

### 2.3 每月真实基础结余是多少

核心公式：

```text
基础结余 = 稳定收入 - 刚性支出 - 日常可控支出 - 债务还款
```

这个数比总收入减总支出更重要，因为原始账本混入了投资、资产购入、信用卡还款和内部转账。

### 2.4 未来 3 / 6 / 12 个月会不会断现金流

核心公式：

```text
未来现金余额 = 当前现金 + 未来收入 - 未来刚性支出 - 已知债务还款 - 已知大额支出
```

输出：

- safe：安全
- watch：观察
- tight：偏紧
- danger：危险

### 2.5 一个大额决策能不能做

输入：

- 支出金额
- 支出时间
- 是否分期
- 分期期数
- 是否影响收入
- 是否可撤销

输出：

- 是否建议执行
- 执行后现金安全垫剩几个月
- 哪个月风险最高
- 是否需要等收入到账
- 是否需要减少其他支出

## 3. 用户范围

第一版只服务用户本人。

所以不做：

- 多用户
- 登录
- 云同步
- 权限系统
- 银行自动连接
- 微信/支付宝自动连接

第一阶段最重要的是把真实财务模型跑准，不是做通用产品。

## 4. 第一版技术形态

推荐路径：

```text
本地 Web App + SQLite + CSV 导入 + 自动分类 + 现金流仪表盘
```

第一阶段先 CLI 跑通数据模型，再做 Web 仪表盘。

## 5. 核心页面

### 5.1 首页仪表盘

展示：

- 本月真实收入
- 本月真实支出
- 本月基础结余
- 当前现金安全月数
- 未来 3 个月风险等级
- 最大现金流压力月份

### 5.2 月度现金流

按月展示：

- 收入
- 刚性支出
- 日常支出
- 债务还款
- 投资
- 资产购入
- 基础结余
- 总现金流

### 5.3 分类审核

只审核需要人工介入的交易：

- unknown
- confidence < 0.8
- 大额交易
- 疑似内部转账
- 疑似债务
- 疑似资产事件

### 5.4 资产和债务事件

展示：

- 买车
- 买房
- 借款
- 还款
- 投资
- 卖资产

### 5.5 决策模拟器

输入计划支出，输出风险等级、最低现金余额、安全月数和建议。

## 6. 账本特殊规则

### 6.1 2021、2022 不进入日常消费模型

这两年数据不完整，主要是房贷和借入相关。

处理：

```text
financial_type = historical_debt_asset_event
```

只参与历史债务和资产事件分析，不参与消费习惯分析。

### 6.2 2023 起进入日常现金流模型

建议：

- 2023：早期参考
- 2024：完整年度基线
- 2025：重点分析年度
- 2026：当前年度跟踪

### 6.3 信用卡还款默认 neutral

信用卡还款不能直接算支出，否则会重复计算。

### 6.4 投资不算日常消费

基金、股票、USDT 等进入 investment_outflow / investment_inflow。

### 6.5 特斯拉购车属于资产事件

不能归入普通交通消费。

### 6.6 卖二手产品属于一次性收入或资产出售

不能算稳定收入。

## 7. MVP 成功标准

第一版成功标准不是界面好看，而是能回答：

1. 每月基础结余是多少。
2. 现金流是否安全。
3. 债务压力占收入多少。
4. 未来 3 到 6 个月有没有断流风险。
5. 大额决策能不能做。
6. 如果不能，差多少钱、需要等到什么时候。
